# 球隊例賽報名(LIFF 網頁版)部署教學

全程免費。你會用到三樣免費服務:**Google 試算表 + Apps Script**(存名單)、**LINE Developers**(身分綁定)、**Netlify 或 GitHub**(放網頁)。

照順序做即可,大約 30–40 分鐘。準備兩個檔案:`Code.gs`、`index.html`。

---

## 流程總覽

```
① 後端(Google)→ 拿到 GAS_URL
② 先把網頁放上線 → 拿到網址
③ 建 LINE Login + LIFF(用②的網址)→ 拿到 LIFF_ID 和分享連結
④ 把 GAS_URL、LIFF_ID 填進 index.html → 重新上傳
⑤ 自己設為管理員、填活動資訊 → 把分享連結貼到群組
```

---

## ① 後端:Google 試算表 + Apps Script

1. 開一份新試算表:瀏覽器輸入 **sheets.new**,命名為「球隊報名」。
2. 上方選單 **擴充功能 → Apps Script**。
3. 把編輯器內預設的 `function myFunction(){}` 全部刪掉,貼上 **`Code.gs`** 全部內容,按 💾 儲存。
4. 右上 **部署 → 新增部署作業**。
5. 點齒輪 ⚙️ 選 **網頁應用程式**,設定:
   - 執行身分:**我**
   - 具有存取權的使用者:**任何人**
6. 按 **部署**。第一次會要求授權 → 選你的 Google 帳號 → 出現「Google 尚未驗證這個應用程式」時,點 **進階 → 前往「球隊報名」(不安全)→ 允許**。(這是你自己寫的程式,安全。)
7. 複製顯示的 **網頁應用程式網址**(結尾是 `/exec`)。這就是 **GAS_URL**,先貼到記事本。

> 之後若修改 `Code.gs`,要重新「部署 → 管理部署作業 → ✏️ 編輯 → 版本選『新版本』→ 部署」,改動才會生效(網址不變)。

---

## ② 先把網頁放上線(拿到網址)

LIFF 需要先有網址,所以先用「還沒填值」的 `index.html` 上線一次。二選一:

**做法 A — Netlify Drop(最快,免帳號也能試)**
1. 把 `index.html` 單獨放進一個資料夾。
2. 開 **app.netlify.com/drop**,把整個資料夾拖進去。
3. 得到一個網址,例如 `https://golf-team.netlify.app`。(註冊帳號可保住這個網址。)

**做法 B — GitHub Pages**
1. 建一個 repository,上傳 `index.html`。
2. **Settings → Pages**,Source 選 `main` / `root`,儲存。
3. 約一分鐘後得到 `https://你的帳號.github.io/repo名稱/`。

把這個 **網頁網址** 也記下來。

---

## ③ 建 LINE Login 與 LIFF

1. 開 **developers.line.biz**,用你的 LINE 登入。
2. 沒有 Provider 的話先 **Create a new provider**(隨意命名,如「球隊」)。
3. 在該 Provider 下 **Create a new channel → LINE Login**,填:
   - Channel name:球隊報名
   - 地區:Taiwan,其餘照基本資料填即可。
4. 進入剛建好的 channel,切到 **LIFF** 分頁 → **Add**:
   - **Endpoint URL**:貼上 ② 拿到的網頁網址
   - **Size**:**Full**
   - **Scopes**:勾選 **profile**、**openid**
   - 其餘預設,Add。
5. 複製這個 LIFF 的 **LIFF ID**(像 `2001234567-AbCdEfGh`)→ 記下來,就是 **LIFF_ID**。
6. 同一頁的 **LIFF URL**(`https://liff.line.me/你的LIFF_ID`)就是 **要分享給隊員的連結**,記下來。

---

## ④ 填入設定,重新上傳

1. 用記事本打開 `index.html`,找到最上方設定區:
   ```js
   var LIFF_ID = "PASTE_YOUR_LIFF_ID";
   var GAS_URL = "PASTE_YOUR_GOOGLE_APPS_SCRIPT_EXEC_URL";
   ```
2. 分別換成 ③ 的 **LIFF_ID** 和 ① 的 **GAS_URL**(保留引號)。存檔。
3. 重新上傳:
   - Netlify:再拖一次資料夾到同一個專案(或 app.netlify.com/drop)。
   - GitHub:覆蓋上傳 `index.html`。
   - 網址不變,所以 ③ 的 LIFF 設定不用動。

---

## ⑤ 啟用與分享

1. **發布 LINE Login 頻道(重要,否則一般會員無法登入)**:LINE Developers → 你的 LINE Login 頻道 → 頻道頁最上方狀態鈕若顯示「**Developing / 開發中**」,點它 → 跳出「Publish this channel?」按「**Publish / 發布**」,變成「Published / 已發布」。沒發布的話,只有開發者本人能登入,其他會員會看到「400 Bad Request… developing status」。
2. 手機 LINE 裡開啟 **LIFF URL**(`https://liff.line.me/...`),第一次會要求授權,允許即可。
3. 進到畫面後,先設定你的本名,再到最下方點 **「設我為管理員」**。
4. 出現「管理員 · 活動設定」區塊,填好日期、球場、費用、名額等,**儲存活動資訊**。
5. 把 **LIFF URL** 貼到球隊 LINE 群組;或用系統內「📤 分享到群組」直接送出例賽卡/報名狀況卡(見下方「分享卡」設定)。

**分享卡(例賽卡 / 報名狀況卡)一次性設定**
- LINE Developers → 你的 **LINE Login 頻道** → **LIFF 分頁** → 找到 **shareTargetPicker** 開關 → 開啟並同意「Agreement Regarding Use of Information」(每個頻道同意一次,對該頻道所有 LIFF 生效)。
- 沒開的話,系統內「📤 分享到群組」按下去會提示尚未啟用。分享只能在 **LINE App 內**操作,且選擇器**不含 OpenChat(公開聊天)**。

**新會員設定(選用)**
- 在「會員名單」分頁加一欄「**會員身份**」,新加入、還沒有差點的人填「新會員」。系統會在他**第二場**成績時,用前兩場平均自動算出差點並納入排名(規則見 README §3)。舊會員留空或填「舊會員」即可。

## ⑥ 進階設定(選用,之後想用再做)

這些不影響基本報名,可日後再開啟。

**球隊小提醒(印在 PDF 右下角)**
- 在 App「管理員專區 → 月例賽成績」面板最下方的「📣 球隊小提醒」輸入框打字 → 儲存即可。
- 一行一條(空白則整塊不顯示)。也可改在試算表 `Config` 新增 `球隊小提醒` 一列、用「｜」分隔多條。
- 若想某場單獨提醒:在「行事曆」該場那列加「備註」欄填字,匯出那場時會優先用它。

**PDF 字級微調**
- PDF 會依人數自動排版壓在單頁 A4。若想固定字級,在 `Config` 新增 `pdfFontPx`(例如 `12`)即可覆寫。

**自動備份**
- 在 Apps Script 編輯器選函式 `setupBackupTrigger` 按 ▶ 執行一次,即會在例賽次日上午自動備份試算表到雲端硬碟「例賽系統備份」(保留最近 30 份)。

**加速登入(建議開)**
- 在 Apps Script 編輯器選函式 `setupKeepWarm` 按 ▶ 執行一次 → 每 5 分鐘輕觸一次,降低 GAS 冷啟動,登入更快。
- 系統還會用「暖快取」:賽前大量隊員登入時,第一位讀過資料後,第二位以後幾乎不必再讀表,登入明顯變快。

**暖快取與「清除快取」**
- 用 **App 操作**的更動(報名、輸入成績、改設定按鈕…)會**立即生效**。
- 但若你**直接在試算表手動改**資料(手改 Config、貼名單、改差點等),最多 **10 分鐘**後才會反映;想立刻生效就按管理員「名單維護(或測試資料)」面板裡的 **🔄 清除快取**。
- 快取存活時間可在 `Code.gs` 最上面的 `CACHE_TTL`(秒,預設 600)調整。

**行事曆欄位**
- 「行事曆」可加「截止日期」欄(報名截止顯示用)、「備註」欄(該場 PDF 專屬提醒)。以欄位標題對應,順序不拘。

**寫入驗章(防偽冒,進階)**
- LIFF 的 Scopes 勾選 `openid`,並在 `Code.gs` 頂端填入 `LINE_CHANNEL_SECRET`(LINE channel 的 Channel secret)。啟用後所有寫入會以 LINE ID Token 驗章取得真實身分,忽略前端帶入值。



- **隊員點了沒反應 / 一直轉圈?** 多半是 GAS 沒設「任何人」存取,或改了 `Code.gs` 沒發布新版本。回 ① 檢查。
- **顯示 LIFF 初始化失敗?** `index.html` 裡的 `LIFF_ID` 沒填對,或 LIFF 的 Endpoint URL 跟實際網址不一致。
- **要換管理員?** 到試算表的 `Config` 工作表,把 `adminUserId` 那格清空,讓新的人重新點「設我為管理員」。
- **下次例賽?** 管理員改好活動資訊後,點「清空整份名單」即可重新開放報名;隊員的全名綁定會保留,不必重設。
- **要改錯成績 / 移除整場?** 直接重送該場成績會自動覆蓋;只改某格用「修改成績」;要整場移除請用「🗑️ 清除本場成績」按鈕(會自動還原差點、清成績/技術獎/PDF)。**請勿手動清空試算表「Scores」分頁**——本場成績另存在 `Config.lastRanking`、差點也需還原,手動清會清不乾淨。
- **大量匯入差點/名單會逾時嗎?** 不會,匯入已批次寫入,40 人約 0.5 秒。
- **手動改了試算表卻沒生效?** 系統有 10 分鐘暖快取。用 App 操作會即時更新;若是直接改試算表,等 10 分鐘或按管理員面板的「🔄 清除快取」即可。
- **PDF 匯出在哪?** 管理員「月例賽成績」面板的「重新匯出 PDF」,或結果頁的「📄 匯出 PDF」;逐場成績可直接開啟各場 PDF。
- **費用:** LINE Login、Apps Script、Netlify/GitHub 這個用量都在免費範圍內。
- **隱私:** 名字、頭像來自 LINE,只有「點開連結」的人會出現在名單;系統看不到沒互動過的群組成員。
