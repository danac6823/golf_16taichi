# 意象太極高爾夫球隊 · 例賽管理系統

LINE LIFF 網頁版的球隊例賽報名 / 分組 / 成績 / 差點 / 會費 / PDF 報表整合系統。
全程使用免費服務:**Google 試算表 + Apps Script**(資料與邏輯)、**LINE Login / LIFF**(身分)、**GitHub Pages / Netlify**(前端網頁)。

> 安裝與上線請見 **`部署教學.md`**。本文件說明系統的**完整功能、資料結構與維運方式**。

---

## 1. 系統架構

| 層 | 內容 |
|---|---|
| 前端 | 單檔 `index.html`(原生 JS LIFF App,高爾夫計分卡風格)。透過 JSONP 呼叫後端。 |
| 後端 | `Code.gs`(Google Apps Script Web App)。`doGet → handle()` 分派各 action,所有資料存在一份綁定的 Google 試算表。 |
| 身分 | LINE Login + LIFF 取得 userId / 顯示名稱;可選用 ID Token 本地驗章防止偽冒。 |
| 報表 | 後端用 `Utilities.newBlob(html).getAs(PDF)` 產生 A4 例賽紀錄,存到雲端硬碟並回傳連結。 |

**關鍵設定**(在 `index.html` 最上方):`LIFF_ID`、`GAS_URL`(結尾 `/exec`)。

## 2. 功能總覽

### 2-1. 會員(隊員)功能
- **一次綁定身分**:首次開啟系統自動帶入你的 **LINE 名稱(唯讀、僅供核對)**,你填**本名(姓名)**、手機、出生年(性別選填)即完成;系統用「姓名」比對「會員名單」主檔,對到→綁定為會員並帶入差點/性別,查無→以來賓加入。之後自動認得。
- **本次例賽資訊卡**:日期、球場、開球時間、費用、截止、名額(資料以「行事曆」分頁為準)。
- **報名 / 請假 / 取消**:一鍵操作;報名可選用餐,請假自動帶不用餐;滿額自動候補。
- **代登記來賓**:可幫自己邀請的來賓報名 / 移除。
- **查看分組(Tee Sheet)**:依開放時間顯示;未到時間顯示「尚未開放」、管理員尚未分組顯示「尚未產生」佔位。
- **查看成績**:本場排名與技術獎(到開放時間才顯示)、逐場成績、年度總表(會員×每月),以及「我的本年度」出席率與獎金累計。
- **整年度賽程**:滿版檢視全年賽程。
- **我的差點 / 會費**:顯示系統管理的目前差點與本年度會費狀態。

### 2-2. 管理員功能(僅 `adminUserId` 本人可見/可用)
- **活動資訊設定**:標題、標準桿、年度區間、幹部、獎金、分組/成績開放時間等(`updateEvent`)。
- **名單管理**:清空整份報名名單(保留身分綁定)、重新開放下一場。
- **自動分組**:依登記順序 / 差點平均 / 二男二女 / 隨機產生分組與開球時間;可重新洗牌、清除(`makeGroups` / `clearGroups`)。
- **輸入成績**:貼上「姓名 OUT IN」自動算總桿/淨桿排名、特殊獎、賽後扣桿並更新差點(`submitScores`);可連同技術獎一起送(`submitMatch`)。
- **改錯重送**:同日期重送會自動覆蓋(還原差點再重算,不重複扣桿)。
- **修改單格成績 / 鎖定本場 / 解鎖**(`lockScores` / `unlockScores`)。
- **清除本場成績(整場移除)**:一鍵還原前三名差點、刪成績與技術獎、清 PDF 連結與本場快取(`clearMatch`)。
- **匯入差點 / 匯入會員名單**(`importHcp` / `importMembers`,皆批次寫入)。
- **技術獎(送球)登錄 / 清除**,送球庫存統計(`recordTech` / `clearTechDate` / `buildBalls`)。
- **會費收款**:整理收款名單、標記已收(`ensurePayList` / `setPaid`;會費明細在 `Payments` 分頁編輯)。
- **行事曆**:建立年度賽程範本(`buildSchedule`)。
- **年度總表 / 成績總表**:寫入試算表分頁(`buildSeason` / `buildSeasonMatrix`)。
- **匯出 PDF 例賽紀錄**(見第 4 節)。
- **球隊小提醒**:在 App 內輸入,會印在 PDF 右下(`setNote`)。
- **年初重設**:依去年度平均重算年初差點、會費歸零(`seasonReset`,需本年度例賽全部結束後才允許)。
- **分享到群組**:用 LINE `shareTargetPicker` 把「🏌️ 例賽卡」或「📋 報名狀況卡」一鍵分享到群組,卡片附「我要報名/請假」按鈕連回系統(需在 LINE 內、且 LINE Login 頻道已啟用 shareTargetPicker)。
- **公告 / 本場注意 / 匯款**:管理員可設三塊內容——📣 最新消息/外地賽(首頁上方常駐)、📌 本場注意事項(報名卡下方,換月自動清空)、💳 匯款資訊(只對「未繳」會員在「我的會費」區顯示,附一鍵複製帳號)。網址自動變連結。
- **代會員報名**:幫「還沒綁定 LINE 的會員」登記出席(下拉挑會員名單、選用餐)。身分算**會員**(不是來賓),成績照會員算、也會進自動分組;他日後綁定 LINE 會自動接上該筆出席(`proxyRegister`/`proxyRemove`,佔位 userId `M:姓名`)。
- **同步名單姓名**:有人誤用 LINE 暱稱綁定時,改好「會員名單」姓名後一鍵把出席名單姓名一併更新(依 LINE 帳號對應,差點也重抓,不必大家重報)(`syncRosterNames`)。
- **匯出出席名單(傳球場)**:純文字含隊名/第X屆、日期/球場/開球、編號名單,結尾帶總幹事聯絡資訊,可貼到 LINE。
- **清除快取**:手動改試算表後,一鍵讓 App 立刻讀到最新(`flushCache`)。
- **測試資料**:一鍵建立 / 清除測試資料(`seedTestData` / `clearTestData`)。

## 3. 排名與差點規則

- `net(淨桿) = round(OUT + IN − 目前差點)`,標準桿 `par` 預設 72。
- **差點來源(三層退路)**:Handicaps「目前差點」→ 空就用「期初差點」→ 都沒有就用「會員名單」的「年初差點」。所以還沒比賽、或還沒匯入 Handicaps 時,也能顯示每人的年初差點。
- **總桿冠軍**:僅會員,本年度一人一次(依年度區間判斷)。
- **淨桿排名**(排除本月總桿冠軍):淨桿低→高;平手序為 後九洞(IN)低 → 差點低 → 年長者得;取前三名。
- **特殊獎**(限非前三):幸運獎=淨桿第 `luckyPlace`(預設 7);逢五跳獎=淨桿第 5/10/15…(`skipStep`);BB 獎=淨桿倒數第二(人數≥5)。優先序 **BB > 幸運 > 逢五**。
- **賽後扣桿**:前三名依差點級距扣;淨桿低於標準桿者前三名再額外扣。新差點四捨五入為整數、最低 0。
- **年初差點** =(去年度平均總桿 − 標準桿)× `hcpResetFactor`(預設 0.85),整數。
- **差點上限**:所有「系統計算」出來的差點(年初重設、新會員建立、賽後扣桿)一律夾在 `0 ~ maxHcp`(預設 **36**)。手動在「會員名單→年初差點」或「匯入差點」填的值不強制夾,由管理員自行維護。

### 新會員(無期初差點)規則
- 在「會員名單」的「**會員身份**」欄標記(值含「新」=新會員;系統用**標題列偵測**該欄,放哪一欄都行)。
- **第一場**:只記總桿,**不算差點、不算淨桿、不列入排名**(畫面顯示「新會員·首場·待第2場」)。
- **第二場**:當即建立差點 = `round( ((第一場總桿 + 第二場總桿) / 2 − 標準桿) × newHcpFactor )`(預設倍率 0.9,且不超過差點上限 `maxHcp`),淨桿 = 第二場總桿 − 此差點,**馬上進入排名**,並寫入 Handicaps(目前+季初)。建立差點的這一場不再額外扣桿。
- **第三場起**:已有差點 → 完全照一般會員規則計算(身份欄可留「新會員」或改「舊會員」皆可,系統以「是否已有差點」為準)。

## 4. PDF 例賽紀錄(2 欄滿版 A4)

- 版面為 A4 直式、四邊 0.8cm 邊界,**依會員人數自動排版**確保壓在單頁:
  - **< 25 人**:成績總表單欄、字級大、保留擊球人數統計列、來賓成績在左欄。
  - **≥ 25 人**:字級依人數自動縮放、移除統計列、來賓成績移到右欄。
- 內容:本月例賽資訊、成績總表(冠軍金底♔、調後差點變動以紅字標示)、桿數排名(含獎金合計)、技術獎送球(含合計)、來賓成績、下次例賽、📣 球隊小提醒。
- 球隊小提醒來源:`Config` 的 `球隊小提醒`(多條用換行或「｜」分隔);若「行事曆」該場有「備註」欄則優先用該場專屬提醒。
- 字級可用 `Config` 的 `pdfFontPx` 強制覆寫微調。
- 每場匯出的連結記在 `Config` 的 `pdfLinks`,於逐場成績可直接開啟。

> 註:GAS 內建 HTML→PDF 轉換器與一般瀏覽器略有差異,首次匯出後可微調 `pdfFontForCount_` 字級對照。

## 7. 效能與穩定性設計

- **請求級快取**:同一次請求內 `Config / 行事曆 / Roster / Members / 會員名單 / 差點` 各只讀一次;寫入對應分頁時即時失效,避免讀到舊值。一次登入(bootstrap)各分頁皆只讀 1 次。
- **跨請求暖快取(CacheService)**:上述資料會存進所有人共用的 CacheService(預設存活 10 分,可改 `Code.gs` 最上面的 `CACHE_TTL`)。賽前大量隊員湧入時,**第一位讀表後,第二位以後幾乎不用再讀表**。App 內任何寫入會「立即清掉對應快取並回填最新」,所以報名/成績即時生效、不會讀到舊資料;只有**直接手動改試算表**才需等 TTL 或按管理員「🔄 清除快取」。
- **登入加速三層**:① 保溫觸發器(`setupKeepWarm`,每 5 分鐘輕觸,降低 GAS 冷啟動)② 前端把 userId 存 localStorage,下次開啟讓 bootstrap 與 LIFF 初始化**並行**起跑 ③ CacheService 暖快取(同上)。
- **出席登記只跑一趟**:報名/請假/取消後端直接回傳最新名單與進度,前端就地更新,不再多打一次 bootstrap。
- **延後載入**:管理員重資料(差點/會費/綁定/球數/技術獎)、年度各場成績、我的統計,皆改為**用到時才抓**,不拖慢登入。管理員與會員登入速度相同。
- **批次寫入**:輸入成績、匯入差點/名單、年初重設、清技術獎、整理收款等,皆**一次讀 + 一次寫**,避免逐列往返(大量匯入不再逾時)。
- **搶報名安全**:報名/分組等寫入使用 ScriptLock,避免同時操作衝突。
- **自動備份**:例賽次日上午自動備份試算表到雲端硬碟「例賽系統備份」,保留最近 30 份(每月一次)。

## 8. 安全性

- **管理員唯一**:`claimAdmin` 只在尚無管理員時可認領;已設定後其他人無法成為管理員;`adminUserId` 僅試算表擁有者可改。
- **寫入驗章**(選用):填入 `LINE_CHANNEL_SECRET` 後,所有寫入動作與管理員資料會用 LINE ID Token 本地驗章取得真實 userId,忽略前端帶入值,防止偽冒。
- **時間鎖**:分組與成績到設定時間才對隊員開放;管理員不受限。

## 10. 維運常見操作

- **開放下一場**:管理員確認行事曆下一場 → 點「清空整份名單」即可重新報名(身分綁定保留)。
- **改錯成績**:直接重送該場成績會自動覆蓋;或用「修改單格成績」;要整場移除用「清除本場成績」。
- **換管理員**:`Config` 的 `adminUserId` 清空 → 對方點「設我為管理員」。
- **球隊小提醒**:管理員專區「月例賽成績」面板輸入即可。
- **不要手動清「成績」分頁**:本場成績另存於 `Config.lastRanking`,且差點需還原;請改用 App 的「清除本場成績」按鈕。

## 11. 部署與更新摘要

1. **後端**:`Code.gs` 貼回 Apps Script → 部署 → 管理部署作業 → ✏️ 編輯 → 版本「新版本」→ 部署(僅存檔不會更新 `/exec`)。時區設 Asia/Taipei。
2. **前端**:`index.html` 重新上傳 GitHub/Netlify,開啟時加 `?v=` 新數字清快取。
3. 分享給隊員的連結:`https://liff.line.me/<LIFF_ID>`(務必在 LINE 內開啟)。

詳細第一次安裝請見 `部署教學.md`。
